Getting Help

If you ever need help while using Git, there is a way to get the manual page (manpage) help for any of the Git commands. You can type git help <command> , where <command> is a git command, for example add, clone, commit, etc.
terminal
git help add
usage: git add [<options>] [--] <pathspec> ... ...

Git Basic

A Git repository is contained in a .git directory, which holds the revision history and other metadata. The directory tracked by the repository, by default the parent directory, is called the working directory. Changes in the working tree need to be staged before they can be recorded (committed) to the repository. Git also lets you restore, previously committed, working tree files.

› Starting git project ( git init )

If you have a project directory that is currently not under version control and you want to start controlling it with Git, you first need to go to that project’s directory, initialize git with command git init.
In this example, we create new folder for our git project.
terminal
mkdir learn-git
cd learn-git
git init
Initialized empty Git repository in /home/user/path/to/learn-git/.git/

› Cloning git project ( git clone )

If you want to get a copy of an existing Git repository — for example, a project you’d like to contribute to — the command you need is git clone <url>. <url> is a path or link that you want to clone from git repository, e.g github, gitlab, etc.
terminal
git clone https://github.com/maaslalani/Codev

› Recording changes ( git add, git status, git commit )

git add is add modified and new (untracked) files to the staging area (a.k.a. the index). You can use git add . for add all modified files

git status The command checks the status and reports that there’s nothing to commit, meaning the repository stores the current state of the working directory, and there are no changes to record.

git commit -m <message> commit the staged changes to the repository with a message which indicates the changes you made.

terminal
touch index.html
git status
On branch master Your branch is up to date with 'origin/master'.
Untracked files:
    (use "git add <file>..." to include in what will be committed)
        index.html
nothing added to commit but untracked files present (use "git add" to track)
git add .
git status
On branch master Your branch is up to date with 'origin/master'.

    (use "git reset HEAD <file>..." to unstage)
        new file: index.html
git commit -m "create index.html file"
git status

    (use "git push" to publish your local commits) nothing to commit, working tree clean

Push repository to github

To put your project up on GitHub, you'll need to create a repository for it to live in. You can visit this page to create repository on github.

After create repository on github, at the top of your GitHub repository's, copy the remote repository URL. for example : https://github.com/maaslalani/Codev.git

Use command git remote add origin <url> to add the URL for the remote repository where your local repository will be pushed. Replace <url> with your github repository's.

terminal
git remote add origin https://github.com/maaslalani/Codev

Verify the new remote URL with command git remote -v

.
terminal
git remote -v
origin git@github.com:maaslalani/Codev.git (fetch)
origin git@github.com:maaslalani/Codev.git (push)

And finally, push your local git repository to github with command git push origin master. You will see that you must include your github account email and password to verifying.

terminal
git push origin master
